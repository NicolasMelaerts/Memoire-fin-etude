⚠️ Deux scénarios de biais

Scénario Standard (90/10) : 90 % des positifs ont une flèche en train, 10 % en test.
Scénario Radical (100/0) : 100 % des positifs ont une flèche en train, 0 % en test. Le biais est maximal — le modèle Normal ne peut pas s’en sortir s’il exploite la flèche.

Lecture FPR / FNR — Le FPR (False Positive Rate, ou taux de faux positifs) FPR = FP / (FP + TN) est la proportion de vrais négatifs que le modèle classe à tort en positif. Dans ce scénario, les négatifs du test portent majoritairement la flèche (≈90 %) : un modèle ayant appris le raccourci « flèche → positif » sera trahi par un FPR élevé.
Le FNR (False Negative Rate, ou taux de faux négatifs) FNR = FN / (FN + TP) est la proportion de vrais positifs que le modèle classe à tort en négatif. Un FNR élevé signale que le modèle rate les positifs sans flèche.

Lecture FPR / FNR — Le FPR (False Positive Rate) FPR = FP / (FP + TN) est la proportion de vrais négatifs classés à tort en positif. Dans le scénario radical, 100 % des positifs ont une flèche en train, 0 % en test : toutes les flèches du test sont donc sur des négatifs. Le FPR devient le révélateur direct du raccourci appris : un modèle qui déclenche « positif » dès qu’il voit une flèche atteint des valeurs catastrophiques.
Le FNR (False Negative Rate) FNR = FN / (FN + TP) est la proportion de vrais positifs classés à tort en négatif. Il reste faible si le modèle sur-prédit le positif (cas typique du piège du raccourci).
