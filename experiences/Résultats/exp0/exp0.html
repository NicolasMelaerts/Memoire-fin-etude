🔬 Expérience 0 : Sélection du λ optimal par grid search

Pour chaque stratégie régularisée, recherche du λ qui minimise la Cross-Entropy sur le set de validation.

📚 Méthodologie

Split du dataset

Le dataset complet est divisé en trois parties :

  • Test (25 % du total) - figé, identique à exp1, jamais utilisé pour sélectionner λ.
  • Train (≈ 56 % du total) - utilisé pour apprendre les poids.
  • Validation (≈ 19 % du total) - utilisé uniquement pour évaluer chaque λ candidat.

Critère et agrégation multi-seed

Pour chaque λ, on entraîne le modèle pendant le nombre d'epochs fixé dans config.py sur plusieurs datasets distincts (un par seed de génération), puis on évalue la Cross-Entropy pure sur le set de validation (sans pénalité, pour ne pas biaiser le choix par la grandeur du terme régularisant). Les courbes affichent la moyenne sur les seeds, la bande grisée représente ±1 écart-type.

Sélection finale par one-standard-error rule

On retient le plus grand λ dont la val loss moyenne reste à au plus un écart-type du minimum observé. Sous l'hypothèse que λ plus grand ⇒ adhésion plus forte aux explications fournies, ce critère privilégie l'explicabilité sans dégrader significativement la performance. Référence : Hastie, Tibshirani & Friedman, The Elements of Statistical Learning, ch. 7.10.

🎯 Sélection finale : λ retenu par one-standard-error rule

Pour chaque stratégie, le plus grand λ dont la val loss moyenne reste à au plus 1 σ du minimum. Ces valeurs seront utilisées dans les expériences finales (chapitres 1-4).
